本来内推的是广州这边的UC震惊事业部，hr说没名额了，就把简历推到了阿里云的计算平台事业部（面试的时候才知道的）


一面（电话面70min）


面试官提前打电话约好时间，还说了要在线编程（编程水平渣渣的靓仔慌的一批。。。）


part1：热身运动


面试官小哥哥人很nice，没有一上来就直接问问题，而是以聊天的形式开始了面试，主要问了什么专业的、实验室做什么的、为什么想做C++开发。。。balabala，这个阶段尽可能多的去说，也是一个引导面试官的过程（后面可能会顺着你说的去问问题）。


part2：手撕代码


// 实现一个函数 能够对输入的字符串数组进行按含有同样的字符进行分类


// 举个例子，输入：``["eat","tea","tan","ate","nat","bat"]``，


// 输出：``[["ate","eat","tea"],["nat","tan"],["bat"]]``


这本来是leetcode 49同字符词语分组原题，后来又加了条件


// 相同的字符只算一次比如 aate 和 ate归为同一类


当时想到的是去重，改了一下程序ok


因为程序中用到了sort()函数，然后让自己实现，其实就是把字符串中字符排序


接着又说到string拷贝过多的问题，有没有其他方法？




还问了其他的，记得不是很清楚了，感觉就是面试官不断换条件变着法考你编程基础。


part3：基础连连问


问题都很基础（可能已经发现我是个菜鸡了），都属于必须掌握的内容。


1、虚函数相关（高频必问）


2、throw异常后没有catch会怎么样


3、死锁相关（高频）


4、进程和线程区别，进程间的通信方式（必问）


5、三次握手（高频必问） 





part4：我问你答


基础问完以后就到了快乐的我问你答环节：你还有没有什么问题？


内心OS：这么好的体（假）现（装）菜鸡虚心好学、追求进步的机会，怎么能错过？


这个环节就当作聊天、向面试官学习的过程，提前准备好问题，放开去问就好。


我觉得问题可以分为两类：


1）、跟面试官相关的


比如：您所在的部门主要是做什么业务的、主要技术栈是什么、如何安排实习生工作的等等，只要不是太涉及个人隐私的都是可以问的；


2）、跟自己相关的





比如：问面试官对自己有哪些学习建议、如何去阅读优秀的开源代码等等；




二面（视频面90min）

11、海量数据处理问题？

						1）、非科班战斗人员一定要提高手撕代码能力，多刷leetcode，刷的时候可以把刷过的试题分类保存到cpp文件中（如链表.cpp），方便以后复习，记不住哪怕背也要背下来！！！
